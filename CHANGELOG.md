# 更新日志

## v0.3.2 - 2026-06-30

### 新增

- **AI Bot 日志系统**：新增仅本客户端可见的内部日志输出，包含「思考」「动作」「感知」「警告」「信息」五个类别，实时显示 AI 的决策过程、目标、动作队列状态。
- **AI 动作序列系统**：AI 现在返回结构化 JSON 指令，包含 `thought`（思考过程）、`goal`（当前目标）和 `actions`（动作列表），支持连续执行多步操作。
- **新增动作类型**：`look`（转向指定角度）、`mine`（挖掘）、`place`（放置方块）、`selectSlot`（选择快捷栏）、`stop`（停止）。
- **动作历史记忆**：记录最近 10 个执行过的动作，随 prompt 一起发送给模型，避免 AI 反复横跳。
- **备用解析机制**：当模型未返回合法 JSON 时，自动回退到关键词匹配解析，保证基础功能可用。

### 改进

- AI 决策 prompt 完全重写，要求模型以纯 JSON 格式回复，不再是简单的关键词匹配，真正让模型「思考」并规划动作序列。
- 控制器 tick 逻辑重构：优先执行动作队列，队列空时才请求新决策，动作执行更连贯。
- 生存系统逃跑时自动清空动作队列，避免冲突。
- 聊天回复和自动决策均接入日志系统。

## v0.3.1 - 2026-06-30

### 修复

- 修复 AI Bot 模式下玩家移动不生效的问题：新增反射修改 `player.input` + 直接设置速度的双重控制机制。
- 修复 AI Bot 连接界面缺少模型名配置的问题，现在连接时必须填写模型名。

### 新增

- AI Bot 生存系统：低血量/着火时自动逃跑（反向移动 + 跳跃），持续 3 秒或直到安全。
- AI Bot 饥饿管理：饥饿值低于 6 时自动查找快捷栏食物并食用。
- AI Bot 危险检测：检测脚下岩浆、火、岩浆块等危险方块。
- AI Bot 聊天过滤：避免回复 AI 自己发送的消息。
- 环境感知增强：新增前方方块检测。

## v0.3.0 - 2026-06-30

### 新增

- **AI Bot 客户端模式**：启动器主菜单新增"AI Bot 客户端"按钮，可输入目标局域网 IP:端口并自动连接。
- **AI 自动控制**：连接成功后，AI 每 3 秒感知环境并自主决策，可执行前进/后退/左/右/停止/跳跃/蹲下/攻击/使用物品/聊天等动作。
- **环境感知**：AI 能感知自身位置、生命值、饥饿值、周围敌对/友好/玩家实体、时间等。
- **自动聊天回复**：当游戏内聊天提到 AI 名字或关键词时，AI 会自动延迟回复。
- **自动攻击**：AI 检测到周围敌对实体时会自动转向并攻击。

## v0.2.1 - 2026-06-29

### 改进

- AI  companion 以玩家形式呈现，移除实体名称标签，使用 `GameProfile` 名字自然显示。
- 新增客户端伤害系统：检测坠落伤害，血量归零后死亡并延迟 3 秒重生。
- 不再强制保持满血，AI 可以真正受到伤害。

## v0.2.1 - 2026-06-29

### 修复

- 修复进入世界时 `openToLan` 触发空指针崩溃的问题，增加 `client.player != null` 检查和 2 秒延迟。
- 修复 `ClientWorld.addEntity(int, Entity)` 在 1.21.1 中不存在导致的编译失败，改为单参数 `addEntity(Entity)`。
- 为实体管理增加全面的 `try-catch` 保护，防止客户端 tick 崩溃。

## v0.2.0 - 2026-06-29

### 新增

- AI 角色（XuanXuan-ZhengGui）加入游戏后自动连接模型，可响应聊天内容。
- AI 角色能自主走动：距离远时传送跟随，距离适中时平滑移动，靠近时减速并面向玩家。
- AI 角色具备基础生存逻辑：保持满血、清除摔落伤害、保持呼吸，不会死亡。
- AI 角色在玩家聊天提到其名字、模型名或"玄玄"/"ai"时，会延迟 0.8~2.3 秒后自动回复。
- 选择"加入游戏"后自动开启局域网世界，其他玩家可直接搜索加入。

### 改进

- 优化 AI 实体跟随逻辑，增加朝向同步和头部转动。
- 聊天触发支持 `@模型名` 和 `@*模型名`，同时在加入游戏模式下监听普通聊天内容。

## v0.1.1 - 2026-06-27

首个可发布版本，面向 Minecraft 1.21.1 Fabric 客户端。

### 新增

- 初始化 Fabric 1.21.1 Mod 项目结构。
- 增加 `M` 键打开已加载地图界面。
- 地图界面中已加载区块显示为绿色，未加载区块显示为黑色，玩家所在区块显示为黄色。
- 在游戏暂停菜单中增加"加入本地或云端AI"按钮。
- 增加 AI 服务选择界面，支持"云端"和"本地"两类入口。
- 云端入口支持输入 API 和模型名。
- 本地入口支持选择 `ollama` 与 `llama.cpp`。
- `ollama` 默认连接 `127.0.0.1:11434`。
- `llama.cpp` 默认连接 `127.0.0.1:8080`。
- 配置完成后支持选择"聊天"或"加入游戏"模式。
- "加入游戏"模式会生成名为 `XuanXuan-ZhengGui` 的 AI 玩家实体，并随机默认皮肤。
- 游戏内支持通过 `@*模型名` 或 `@模型名` 触发 AI 回复。
- 增加本地配置保存能力。
- 增加中英文语言文件。

### 已知限制

- 云端 API 已预留输入与路由入口，但不同云端服务商的请求格式需要按实际协议继续补充。
